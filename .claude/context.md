# 📋 抢场脚本开发进度同步文档

### 1. 项目背景

> 重要！！！：如果对实际的羽毛球场地信息或者预约逻辑不清楚导致影响编码，请立即向我提问！

- **目标**：开发一个基于 Node.js 的自动化脚本，用于抢钉钉工作台内的学校羽毛球场地（H5 页面，只能在手机端打开）。
- **技术路径**：协议模拟（通过直接发送 HTTP/HTTPS 请求实现，效率最高）。
- **当前阶段**：已完成环境搭建与接口分析，准备进入代码编写与逻辑调试。

### 2. 具体场地信息

- 一次最多只能预约一片场地的2个时间段
- 共有12片场地，预约时间段如下：

```js
 "time_list": [
    "08:00",
    "09:00",
    "10:00",
    "11:00",
    "11:40",
    "13:20",
    "14:00",
    "15:00",
    "16:00",
    "17:00",
    "18:00",
    "19:00",
    "20:00",
    "21:00"
  ]
```

- 预约日期、预约场地、预约时间段这三个信息应该由用户决定，可以写在配置文件中

### 3. 已完成的基建

- **抓包环境**：已成功配置 Mac 版 Charles 代理。
- **设备连接**：iPhone 已连接 Mac 代理，并完成了 SSL 证书安装与“始终信任”设置。
- **可行性验证**：已确认可以在MacBook的 Charles 中截获iphone的 HTTPS 加密流量，并能清晰查看到请求的 URL、Headers (Cookie/Token) 以及 Body (JSON 格式参数)。

### 4. 核心情报 (Data Points)

- **核心预约接口**：预约抓包到两个api请求，分别保存在：`./api/create_book_info` `./api/create_order.md`
- **身份凭证 (Auth)**：token保存在`../.env`中，请求api时拼接在header中的Authorization
